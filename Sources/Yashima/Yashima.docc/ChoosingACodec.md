# Choosing a Codec

Yashima stores generated values through explicit codecs. The codec identity is
part of the effective cache entry identity, so the same ``CacheKey`` can safely
store different encoded forms.

## Data

Use ``DataCodec`` for generated bytes that are already in the form you want to
store:

```swift
let data = try await cache.data(for: key) {
    try await renderBytes()
}
```

Use ``CompressedDataCodec`` when the generated value is large text-like `Data`
such as rendered HTML, JSON, manifests, or summaries:

```swift
let documents = cache.using(CompressedDataCodec())

let html = try await documents.value(for: key) {
    Data(renderedHTML.utf8)
}
```

Compression is opt-in. Avoid it for already-compressed payloads such as JPEG,
PNG, and video data unless measurement shows a benefit.

## Codable

Use ``CodableCodec`` for small structured derived values:

```swift
let duration: Double = try await cache.codable(for: key) {
    try await loadDuration()
}
```

The default format is JSON. Binary property lists are also available through
``CodableCodec/Format/propertyList``.

## Images

Use ``ImageCodec`` or the JPEG/PNG convenience methods for generated platform
images:

```swift
let thumbnail = try await cache.jpeg(for: key, quality: 0.85) {
    try await renderThumbnail()
}
```

Image codecs store explicit PNG or JPEG data. Yashima does not download remote
images; use a dedicated image pipeline for that problem.
